Version 1.0

This is the first major version of the Artist Index with all of the basic, main functionality of the index working.

  • Established the structure of artist database and the types of data to store in the database. SQLite was used as the database engine as traffic was not expected to be heavy.
  • Built the frontend using HTML, CSS, and JavaScript. Bootstrap CSS was used for styling the pages and automatically handling desktop/mobile clients.
  • Drew and animated the characters in the background.
  • Built the backend using Python and Flask. This was chosen due to being more familiar with Python.
  • Got the main index page to display the artist database in a table. Used the DataTables JS library for functionality like searching and sorting.
  • Added a user login system. No user registration was implemented on purpose, as logged-in users would be capable of modifying the database.
  • Added functionality for users to add, edit, and remove artists from the database.
  • Added a detailed view for artists which displays additional data. Logged-in users can additionally see hidden data that is not publicly available.
  • Added a control panel for users. The idea behind the control panel is to have a place that has all the administrative controls all in one page.
  • Added a password change form for users in the control panel. New passwords also need to be at least 8 characters long.
  • Added an "about" page, which has a link to the GitHub repository and a button to hide the interface.
