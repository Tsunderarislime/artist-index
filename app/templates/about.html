What is this?

It's a personal project for a web application that hosts a database of artists I follow on various social media platforms. It uses Flask for the web application framework and SQLite as the database engine. I have been hosting an instance of this web application using Railway. You are likely reading this text from that exact instance.

Why did you make this?

The main platform I use to follow artists is Twitter, but due to policies regarding training AI models and unwarranted edits with uploaded works, artists have been making accounts on other platforms. Artists usually do link their other socials on their Twitter profile, but there are cases where their other links are either not linked at all or are nested within another website that contains links to their other socials.

I made this web application so that I compile a list of the socials of the artists I follow, all into a single website that I can access anytime. It's kinda like an address book stored in the cloud, but I get full control over the functionality in the backend and the user interface in the frontend.

Where is user registration?

The user functionality is not designed to be available to the public in this web application. It's designed to act as a way for me and trusted users to run administrative tasks without having to SSH onto the server. The only way to register users right now is to SSH onto the server and manually add a new user with SQLAlchemy. There are no plans to open user registration to the public.
